## Library inconsistency: ground-floor layer ordering

`nza_engine/library/constructions.py:_construction()` documents the
layer convention as 'ordered outside to inside'. Walls and roofs follow
this consistently. Ground floors do not: `ground_floor_slab.outside_layer`
is `GFloor_Std_Carpet` (the **indoor** finish), with the XPS insulation as
the last layer. EnergyPlus's U-value calc is direction-symmetric so the
reversed order produces the documented U=0.22 W/m²K — but transient
behaviour, sol-air, and any code that assumes the layer convention will
misread these constructions.

The Part 1 audit script compensates by reading the construction `type`
field and inverting the inside/outside split for `type=floor`. The library
data itself is **not** modified here — see `docs/state_1_divergences.md` §6.
